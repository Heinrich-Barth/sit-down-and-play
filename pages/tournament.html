Tournament Information

Tournament games require a mechamism to verify that each participant always plays with the identical deck.

A result token utilises stores the following pieces of information:
1. Date of Game
2. Duration and number of turns played
3. Player names and points scored (i.e. results)
4. Deck checksums

Check game result token

Understanding the result token

A result token utilises the JSON Web Token standard to store game information. The token is also signed by so the information cannot be changed.

The deck checmsum is a unique hash generated from the deck each player played with. Changing a deck in any form will automatically result in a different checksum. Thereby, the tournament organisers can verify that a deck is identical throughout the tournament.

How to obtain the result token

Therefore, when you have finished your game, a special result token will be created that contains all relevant game data and also a signature which guarantees its authenticity.
You can copy the result token via the Copy result to clipboard function automatically. If this fails, the token will be displayed and you can copy it manually.
